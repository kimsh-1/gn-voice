Claude Code에는 서브에이전트라는 물건이 있어서
이걸 잘 쓰면 반복 작업을 통째로 넘겨버릴 수 있습니다

서브에이전트가 뭐냐고요?
메인 세션이랑 완전히 분리된 컨텍스트에서 따로 일하는 보조 에이전트인데, 메인 컨텍스트를 안 더럽히면서 무거운 작업을 던져둘 수 있다는 것, 이게 제일 큰 장점이죠

우선 에이전트 정의 파일부터 만들어야 하는데
프로젝트 루트의 `.claude/agents` 디렉토리에 마크다운 파일을 하나 만들어 두고, 파일 맨 위에는 YAML 프론트매터로 `name`이랑 `description`을 적습니다. 이때 `description`에는 트리거 조건을 분명하게 담아야 하고요. 설명이 흐리멍덩하면 에이전트가 아예 호출이 안 됩니다. 모호하게 써두면 안 불려요

그다음이 시스템 프롬프트입니다
여기서 챙길 게 셋인데
하나, 에이전트가 뭐 하는 녀석인지 한 문장으로 못 박고, 둘, 입력이랑 출력 형식을 구체적으로 적고, 셋, 도구 쓰는 횟수에 상한을 겁니다. 제한을 안 걸면 이 녀석이 쓸데없는 탐색으로 토큰을 줄줄 흘리거든요

자, 실제로 한번 돌려볼까요?
예시로 만들어 둔 `code-reviewer` 에이전트한테 파일 하나 던져서 리뷰시켜 보면 약 30초쯤 지나 결과가 툭 돌아오는데, 같은 걸 메인 세션에서 하면 컨텍스트가 5,000토큰 넘게 깎이고, 서브에이전트로 돌리면 메인이 먹는 건 결과 요약분 200토큰 수준. 그게 다예요

마지막으로 주의할 게 하나 있는데
서브에이전트는 세션 상태를 공유하지 않아서, 필요한 정보는 전부 프롬프트에 대놓고 다 넣어줘야 합니다. 경로는 무조건 절대 경로로 쓰고, 기대하는 출력 형식도 같이 적어서 넘기세요

결국 서브에이전트는 컨텍스트를 격리하려고 쓰는 도구고, 정의 파일이랑 시스템 프롬프트 두 개로 굴러가는데, 트리거를 얼마나 분명하게 쓰느냐 도구 제한을 거느냐가 품질을 가릅니다
