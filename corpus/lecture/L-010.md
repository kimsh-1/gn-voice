## 핵심 원칙: 좌표를 정확히 찍어라

```
프롬프트 최적화 = 임베딩 좌표 최적화

모호한 표현 → 좌표 분산 → 평균적 결과
정확한 표현 → 좌표 집중 → 전문적 결과
```

## 원자 노트 맵

### 문제 진단: 왜 결과가 이상한가?
- [[모호한-프롬프트의-좌표-분산-문제-2026-02|모호한 프롬프트의 좌표 분산 문제]]
  - "비밀 계약" → 좌표가 여러 도메인에 분산
  - "NDA" → 법률 클러스터에 정확히 착지
  - AI가 "지맘대로" 말하는 진짜 이유

### 해결책 1: 구체성 높이기
- [[프롬프트-구체성과-문제공간-축소-2026-02|프롬프트 구체성과 문제 공간 축소]]
  - "코드 짜줘" vs "REST API 구현해" — 천지 차이의 원인
  - 전문가 vs 일반인의 핵심 차이 = 좌표 정확도
  - 구체성 스펙트럼: 매우 낮음 → 매우 높음

### 해결책 2: 한영 하이브리드
- [[한영-하이브리드-프롬프트-전략-2026-02|한영 하이브리드 프롬프트 전략]]
  - 영어 전문 용어 (좌표 정확도) + 한국어 맥락 (뉘앙스)
  - "PRD 작성해줘 + 한국 스타트업 스타일로"
  - 실전 적용 패턴 3가지

### 배경 이해: 한국어의 현실
- [[한국어-AI-학습데이터-편중과-임베딩-정확도-2026-02|한국어 AI 학습 데이터 편중과 임베딩 정확도]]
  - 영어 60%+ vs 한국어 2-3% (학습 데이터 비율)
  - 한국어 전문 용어의 임베딩이 덜 정교한 이유
  - 대응 전략: 한영 하이브리드, 영어 병기, 도메인 프라이밍

## 즉시 적용 가능한 체크리스트

### Before → After 패턴

| # | Before (모호) | After (구체) |
|---|--------------|-------------|
| 1 | "코드 짜줘" | "REST API 엔드포인트 구현해" |
| 2 | "비밀 계약 설명해줘" | "NDA의 법적 구속력 분석해줘" |
| 3 | "이메일 써줘" | "Re-engagement 이메일, 30일 미접속 유저 대상" |
| 4 | "보고서 써줘" | "PRD 작성해줘, 한국 스타트업 맥락" |

### 실천 원칙

1. **전문 용어 우선**: 일상어 대신 도메인 용어 사용
2. **영어 핵심어 + 한국어 맥락**: 하이브리드 전략
3. **범위 지정**: 포함/제외 항목 명시
4. **형식 강제**: 출력 형식 사전 지정
5. **생성 → 검토 → 수정 루프**: 한 번에 완성하려 하지 않기
