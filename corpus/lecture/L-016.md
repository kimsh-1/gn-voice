# 실전 가이드: 피해야 할 표현과 좋은 습관

> "미사여구를 멈추고, 조건을 추가하십시오"

## 프롬프트를 망치는 나쁜 습관들

### 1. "알아서 잘/적당히" 사용

```markdown
❌ 문제: 목표와 성공 조건을 비움
   "알아서 잘 해줘"
   
→ 결과: 모델은 빈칸을 자기 방식으로 채움
→ 결과는 사용자의 기대에 못 미침
```

**"알아서"는 빈칸을 AI 맘대로 채우라는 뜻**

### 2. "좋게/깔끔하게/고급스럽게" 사용

```markdown
❌ 문제: 평가 기준이 없음
   "깔끔하게 정리해줘"
   
→ 결과: 모델은 평균적인 톤 글을 쓰거나 과장되게 씀
```

**기준이 모호하면 결과도 모호하다**

### 3. "자세히" 남용

```markdown
❌ 문제: 어디까지를 명백히 지정하지 않음
   "자세히 설명해줘"
   
→ 초보자용의 "자세히"와 전문가용의 "자세히"는 명백히 다름
```

**전문가의 "자세히"는 다르다** - 구체적 범위 지정 필요

### 4. "최대한/완벽하게" 사용

```markdown
❌ 문제: 기준이 없는 것과 마찬가지
   "최대한 완벽하게 작성해줘"
   
→ 신입사원에게 "완벽하게 해와"라고 해서 잘 나올 확률은?
```

**"완벽하게"는 기준이 아니다**

### 5. "대충" 허용

```markdown
❌ 문제: 품질 하한선을 지움
   "대충 요약해줘"
   
→ 사용자가 "대충"을 허용하면 AI도 적당히 토큰을 아낌
```

**AI는 잘 조져야 한다** - 품질 하한선을 명시해야 함

## 금지어 총정리 (Style Faux Pas)

| 금지어 | 문제점 | 대안 |
|--------|--------|------|
| 알아서 잘 | 목표 부재 | 구체적 성공 조건 명시 |
| 깔끔하게 | 기준 모호 | 형식과 구조 지정 |
| 대충 | 품질 하한선 붕괴 | 최소 품질 기준 명시 |
| 자세히 | 범위 불명확 | 대상 독자와 깊이 지정 |
| 완벽하게 | 기준 없음 | 체크리스트 형태로 기준 제시 |
| 적당히 | 양적 기준 없음 | 수량, 길이 명시 |

> **핵심**: 미사여구를 고치지 말고, **조건을 추가**해야 한다

## 생성→검토→수정 루프

### 한 번에 완성하려는 태도가 가장 비싸다

> "AI 사용에서 가장 비싼 습관은 한 번에 완성하려는 태도"

- 가장 비싼 재료 = **시간**
- 모델은 확률적으로 문장을 생성
- 작은 표현 차이에 민감

**"고쳐줘 고쳐줘"를 반복 = AI에게 끌려다니는 것**

### 품질 수렴 루프

```
정답을 맞히는 게임 ❌
품질을 수렴시키는 공정 ✅
```

**5단계 루프**:

```
(1) 프롬프트 작성 후 생성
        ↓
(2) 기준 미달이라면 새 세션에 수정 후 생성
        ↓
(3) 결과를 비교해 부족한 조건을 추가
        ↓
(4) 검토 지시로 결함을 찾음
        ↓
(5) 수정 지시로 재생성
        ↓
    (반복)
```

### 루프 안에서 해야 할 것

1. **단어의 관계성 찾기**: 어떤 단어가 효과적인지 파악
2. **어디까지 쓸지 판단**: 적절한 지점에 도달하면 사용
3. **결과물 미리 구상**: 어떻게 나올지 생각 안 하면 공허한 결과

> "결과물을 구상해야 한다. 어떻게 나올지 생각 안 하면 공허한 이야기가 나온다"

## 단일 프롬프트의 종말

### 에이전트 시대의 도래

> "단일 프롬프트 하나의 가치는 이제 거의 없다"

- 2025년부터 계속 언급되던 흐름: **에이전트의 시대**
- 프롬프트를 공개해도 괜찮은 이유
- 단일 프롬프트보다 **루프와 시스템**이 중요

### 프롬프트 공개에 대한 질문

"이렇게 프롬프트 다 알려주셔도 되나요?"
"프롬프트를 이렇게 공개하면 누가 그대로 베끼지 않나요?"

**답**: 프롬프트 엔지니어링은 중요하지만, **단일 프롬프트의 가치는 이미 희석**되었다

## 결론: AI의 네비게이터가 되라

### 당신은 AI의 네비게이터

> "AI라는 거대한 도서관에서 원하는 책을 찾으려면, '좋은 책 줘'가 아니라 **'3열 5번 책장'**이라는 정확한 좌표를 주세요"

- 좌표 = 여러분이 쓰는 **키워드**
- 도메인 특화 = 자기한테 맞는 옷을 입는 것
- **업무하는 것처럼** AI를 써야 한다

### 최종 체크리스트

프롬프트 작성 전 자문:
- [ ] 내가 하는 방식이 맞는가?
- [ ] 아니면 "해줘" 프롬프팅인가?
- [ ] 결과물을 미리 구상했는가?
- [ ] 조건이 명확한가?
- [ ] 루프를 돌릴 준비가 되어 있는가?
