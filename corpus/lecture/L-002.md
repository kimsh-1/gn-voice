# 당신의 AI가 말을 안 듣는 이유 – [심화 GPT Memory 제대로 쓰기]

## 개요

총합 좋아요 800개 이상을 받은 GPT 활용법 특집편

GPT를 써도 대답이 달라요. 이는 보통 메모리가 서로 다르기 때문인데요.

GPT의 경우 **메모리를 최적화 하냐 안하냐로 성능이 정말 많이 달라집니다.**

**[참고: 어렵습니다. 그래도 난 GPT 제대로 쓸꺼다 하시는분에게 추천]**

## 0. 그래서 진짜 달라요??

맨처음 글에서 앵커링과 베이시스에 따라서 달라진다 했죠. 먼저 한번 비교해 볼까요?

간단하게 **"AI가뭐야"** 라는 질문을 한다고 해보죠.

**비교 실험:**
- 1번: 임시모드 [튜닝 아예 없음]
- 2번: 제가 튜닝해논 메모리
- 3번: 프롬프트 생성기로 1차 편향성을 넣어준 녀석

꾸준히 말씀드리지만 **이런 차이가 발생하는데...**

**이걸 어떻게 효율적으로 써야할까요?**

## 1. Working Memory (세션내 대화)

> 이건 재설명이니 이전 글을 읽으신 분들은 넘기셔도 됩니다.

이전 글에서 설명했지만 **초기 Anchoring과 Basis가 세션에서 대화하는데 영향을 받아요.**

즉, **출발점과 기준을 제대로 잡는 게 정말 중요합니다.** 결국 "내가 뭘 원하는지"를 명확히 알려줘야 한다는 거죠.

### 문제는 어렵죠?

책을 드리긴했는데 어려우시면 **아래글에 포함된 프롬프트 생성기를 한 번 사용해보세요.**

프롬프트 만들어달라 입력해보면, **아래처럼 마크다운 형식으로 구조화된 프롬프트를 받을 수 있어요.**

솔직히 말씀드리면 저렇게 안짜면, 좀 맘에 안듭니다. 안넣으면 나머지를 적당히 설정하거든요.

솔직히 말하면, 아래 요소 중 절반 정도는 지워도 됩니다. 다만 그건 효율화의 영역이고 거기는 나중글에서 다뤄보죠.

### 핵심 포인트

**여기서 이해하셔야 할 핵심은:**

> 아, 세션내 대화는 Working Memory구나

**이거만 기억하세요.**

### 관련 글

**[1편: Anchoring & Basis]** - "당신의 AI가 말을 안 듣는 이유"

"왜 내말을 못알아먹어?"

GPT에게 질문을 했는데 긴 대화에서 느려지고 화난적 있죠?

"대화를 너무 길게 가져가면" "AI는 오히려 멍청해 집니다"

특히 **새로운 일을 시킬 땐 반드시 새 창에서 대화해야 이유가 있어요.**

이유가 뭘까요?

## 2. Persistent Memory

GPT 내에서 메모리는 크게 **2가지**가 있습니다.

### 메모리 종류

1. **Working Memory** - 세션내 대화
2. **Persistent Memory** - 사용자 프로필 형태로 저장되는 설정값

이전 설명한 **엥커링 베이시스가 영향을 받는쪽이 Working Memory**고,

**Persistent Memory는 GPT 내에 2종류가 있어요** [물론 이론적으로 들어가면 훨씬 많지만 일단 간단히]

### GPT 설정창

GPT 설정창을 보시면 아래 **2종류**가 있는거 보이시죠???

- **저장된 메모리**
- **채팅기록 참고**

이 두 가지를 관리해야 제대로 쓸 수 있죠.

**어떻게요??**

---

## STEP 1. 저장된 메모리 관리

### 일단 제 저장된 메모리를 보실까요??

위사진에서 **"메모리 관리하기"**를 눌러보세요.

저야 워낙 작업을 많이해서 이것저것 많지만, **핵심은 자주 사용하는게 기록되어 있다는 겁니다.**

### 문제점과 해결책

**문제는 이상한게 등록되는 경우도 있어요.** 그래서 저는 주마다 필수적인건 제외하고 지워버립니다.

한번 쭉 읽어보시면서:
- "이건 기억해줬으면 좋겠다"
- "이건 필요없다"

가 보일꺼에요.

### 관리 방법

아니면 GPT에게:

> "저장된 메모리중에 나한테 필요없는게 뭐야"

한번 물어보시고 필요없는건 지워버리시면 됩니다.

**어차피 "기억해" 라고 명령하면 다시 들어가니 편하게 하세요.**

## STEP 2. Implicit Memory

### Implicit Memory란?

다음 프롬프트를 입력해 보세요:

```
#상황
GPT의 메모리를 최적화 하려해

#지시
Implicit Memory에 기억된 정보를 나열해줘
```

이건 **반복된 대화 흐름에서 자동 추론된 정보**로 GPT가 기억하고 있는 녀석들 입니다.

### 왜 중요한가?

**중요한건 이거에도 응답에 영향을 받는다는거에요.**

막 갑자기 이상한 예시를 들고오면 거의 이거때문에 꼬여서 이상한 대답을 한다 보시면 됩니다.

### 관리 방법

결국은...? **이걸 또 관리 해줘야합니다.**

관리하시려면 아래 프롬프트를 입력하세요:

```
XX는 잊어버려
이거 말고 다른정보도 보여줘
```

**이건 직접 제어 해야합니다.**

### 핵심 포인트

> 보통 "내꺼 GPT가 이상하다?" 거의 여기서 꼬인거에요

### GPT 메모리 최적화 체크리스트

1. **Working Memory 관리**
   - 초기 Anchoring과 Basis 설정
   - 구조화된 프롬프트 사용
   - 새 작업 시 새 창 사용

2. **Persistent Memory 관리**
   - 저장된 메모리 주기적 점검 (주간)
   - 불필요한 메모리 삭제
   - GPT에게 필요없는 메모리 물어보기

3. **Implicit Memory 관리**
   - 자동 추론 정보 확인
   - 잘못된 추론 정보 직접 제어
   - 이상한 답변 시 우선 점검 대상

4. **맞춤형 지침 설정**
   - 자동 생성 프롬프트 활용
   - 대화 품질 최적화
   - 개인화된 GPT 경험

### 핵심 메시지

> GPT의 메모리를 최적화 하냐 안하냐로 성능이 정말 많이 달라집니다.
