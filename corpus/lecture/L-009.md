# LLM 내부 동작 원리

> **프롬프트가 AI 내부에서 어떻게 처리되는지 — 토크나이저부터 어텐션까지**

[[AI-프롬프트-테크닉-시리즈-마지막편-프롬프트기법총정리-MOC|← 메인 MOC로 돌아가기]]

---

## 처리 파이프라인 개요

```
텍스트 입력
    ↓
1. 토크나이저 (BPE) — 텍스트를 토큰 조각으로 분할
    ↓
2. 임베딩 레이어 — 각 토큰에 고차원 벡터 좌표 부여
    ↓
3. 어텐션 메커니즘 — 토큰 간 관계 가중치 계산
    ↓
4. 다음 토큰 확률 계산
    ↓
출력 생성
```

프롬프트에 쓰는 **단어 하나하나**가 이 파이프라인에서 다르게 처리됨.
전문 용어 vs 일상어의 결과 차이가 발생하는 근본 원인.

---

## 원자 노트 맵

### 1단계: 토크나이저
- [[BPE-토크나이저와-한국어-토큰-비용-2026-02|BPE 토크나이저와 한국어 토큰 비용]]
  - BPE (Byte Pair Encoding) 알고리즘 원리
  - 한국어 교착어 특성 → 토큰이 더 잘게 쪼개짐
  - 비용 차이: 같은 의미라도 한국어가 토큰 비용 더 높음

### 2단계: 임베딩
- [[임베딩-벡터-좌표와-의미-공간-2026-02|임베딩 벡터 좌표와 의미 공간]]
  - 384차원 벡터 좌표 = 단어의 의미적 위치
  - 비슷한 의미의 단어 → 가까운 좌표
  - 프롬프트의 단어 선택 = AI에게 주는 좌표

- [[Word2Vec-의미-연산과-벡터-산술-2026-02|Word2Vec 의미 연산과 벡터 산술]]
  - King - Man + Woman = Queen (벡터 연산)
  - 의미 클러스터: 관련 개념들의 군집
  - 전문 용어 → 클러스터 중심부에 착지

### 3단계: 어텐션
- [[어텐션-메커니즘과-전문용어의-중력-2026-02|어텐션 메커니즘과 전문 용어의 중력]]
  - 어텐션 = 어떤 토큰을 더 강하게 참고할지 가중치 계산
  - 전문 용어의 "중력": 관련 개념에 레이저처럼 집중
  - 모호한 단어 → 어텐션 분산, 전문 용어 → 어텐션 집중

## 핵심 인사이트

### 왜 같은 질문에 다른 결과가 나오는가?

```
같은 주제라도:
├── 어떤 단어를 먼저 쓰는가 → 위치 편향
├── 어떤 단어를 선택하는가 → 임베딩 좌표
├── 어떤 형식을 강제하는가 → 어텐션 방향
    ↓
모델이 참고하는 단서의 경로가 완전히 달라짐
```
