# 트랜스포머와 벡터 스페이스 원리

> "AI는 정답 자판기가 아닙니다"

## 핵심 개념: AI는 어떻게 동작하는가

### AI는 정답 자판기가 아니다

```
❌ 잘못된 이해:
질문 → [저장소에서 정답 검색] → 정답 출력

✅ 올바른 이해:
입력 → [다음 단어 확률 계산] → 가장 확률 높은 단어 이어붙이기
```

- AI는 질문을 듣고 저장소에서 정답을 꺼내는 자판기가 **아니다**
- 입력된 문맥을 보고 **"다음에 올 가장 확률 높은 단어"**를 이어 붙이는 것
- 이 구조를 **트랜스포머(Transformer)**라고 부른다

### 트랜스포머의 핵심

> "벡터를 끌어오듯 문맥을 재조합하는 것"

- 입력이 들어오면 → 그 입력을 바탕으로 다음 단어가 무엇일지 확률을 만든다
- 같은 요청이라도 다음 요소에 따라 성능이 올라갈 확률이 높다:

| 제약 조건 | 효과 |
|----------|------|
| 1. 무엇을 포함하고 무엇을 제외하는지 | 범위 지정 |
| 2. 어떤 형식으로 내야 하는지 | 형식 강제 |
| 3. 틀리면 어떤 종류의 피해가 생기는지 | 오류 방지 |

## 토큰과 임베딩 벡터

### 텍스트 처리 과정

```
텍스트 입력
    ↓
토큰으로 쪼개기 (Tokenization)
    ↓
토큰을 임베딩 벡터로 변환 (Embedding)
    ↓
벡터 공간에서 연산
    ↓
출력 생성
```

AI는 단어를 그대로 읽지 않는다:
1. **토큰화**: 텍스트를 토큰으로 쪼갠다
2. **임베딩**: 토큰을 임베딩 벡터로 바꾼다
3. **연산**: 벡터 공간에서 계산한다

### 전문 용어의 힘

> "특정 도메인이나 실무에서 통용되는 용어는 모델이 학습 과정에서 반복적으로 학습된 데이터"

- 전문 용어를 정확히 쓰는 순간 → 모델은 **더 좁은 의미 공간**을 가져온다
- 쉽게 말하면: "AI가 학습된 데이터 중 **특정 키워드를 쓰면 성능이 잘 나온다**"

## Attention 메커니즘

### 데이터를 끌어오는 방식

> "입력된 문장 안에서 어떤 단서를 더 강하게 참고할지 계산해 재조합"

- 학습된 데이터 내에서 벡터로 계산해 특정 출력물을 만든다
- 그 단어들이 **공간 벡터를 끌어온다**는 표현이 적절

### 입력이 달라지면 경로가 달라진다

```
같은 주제라도:
├── 어떤 단어를 먼저 주는지
├── 어떤 문장을 강조하는지
├── 어떤 형식을 강제하는지
    ↓
모델이 참고하는 단서의 경로가 바뀜
    ↓
경로가 바뀌면 결과가 바뀜
```

- **키워드가 잘 먹힌다**는 표현의 근거
- 특정 문장에 있는 **로직 자체를 끌어온다**고 봐도 무방

## 초보자 vs 전문가 비교

### 문제 공간의 차이

| 구분 | 일반인 | 전문가 |
|------|--------|--------|
| 표현 | "자세히 설명해줘" | 업계 용어 사용 |
| AI의 탐색 범위 | 넓은 들판 | 좁은 문제 공간 |
| 결과 품질 | 평균적인 답변 | 높은 퀄리티 |

### 연산량 차이

페르소나가 형성될 때:
1. **전문가 중심 글**: 한 번 연산하는 데 드는 양이 낮음
2. **아무것도 모르는 일반인에게 설명 + 퀄리티 높은 결과물**: 연산량이 높음

> "전문가가 쓰는 특정 단어 조합이 강해지는 이유는 **정보를 더 많이 담아서가 아니라 정보의 좌표를 더 정확히 찍어서**"

## 실무 적용 포인트

### NotebookLM의 원리

- NotebookLM이 고평가 받는 이유 = 주어진 소스 기반으로 생성
- **적절한 응답을 만드는 복잡한 과정을 생략**할 수 있음
- 반대로 말하면: **제대로 제약조건을 쓰면 NotebookLM과 같은 효과**를 낼 수 있다

### 제약 조건 설계 원칙

```markdown
효과적인 제약 조건:
1. 범위 지정 - 무엇을 포함/제외할지
2. 형식 강제 - 어떤 형식으로 출력할지
3. 품질 기준 - 틀리면 어떤 문제가 생기는지
```
