# 프롬프트 단어의 5가지 역할

> "키워드를 왜 써야 하는지, 그 단어가 어떤 역할을 수행하는지 생각해야 한다"

## 핵심 원칙

**"전문가 키워드 리스트를 외워라"가 아니다**

- 키워드를 **왜 써야 하는지** 이해해야 한다
- 그 단어가 **어떤 역할을 수행하는지** 파악해야 한다
- 단순 키워드가 아닌 **제대로 된 로직**까지 들어가야 한다

## 프롬프트 단어의 5가지 역할

AI가 문장을 해석할 때, 단어들은 다음 5가지 역할로 작동한다:

### 1. 범위 지정 (Target Scope)

```markdown
역할: AI가 탐색할 문제 공간을 좁힌다

예시:
❌ "마케팅에 대해 알려줘"
✅ "B2B SaaS 스타트업의 콘텐츠 마케팅 전략"
```

- 무엇을 포함하고 무엇을 제외할지 명시
- 도메인 특화 용어로 범위를 좁힘

### 2. 목적 고정 (Goal)

```markdown
역할: AI가 달성해야 할 명확한 목표를 설정

예시:
❌ "좋은 이메일 써줘"
✅ "고객 이탈 방지를 위한 재활성화 이메일, 목표: 7일 내 로그인 유도"
```

- 성공 조건을 명확히 정의
- "알아서 잘"은 목표 부재

### 3. 산출 형식 강제 (Format)

```markdown
역할: 출력물의 구조와 형태를 지정

예시:
❌ "정리해줘"
✅ "요약-본문-결론 3단 구조, 각 섹션 글머리 3개 이내"
```

- 표, 리스트, JSON 등 구체적 형식 지정
- 길이, 섹션 구조, 스타일 명시

### 4. 오류 금지 (No-Go)

```markdown
역할: 하지 말아야 할 것을 명시해 품질 하한선 설정

예시:
✅ "추측성 정보 사용 금지"
✅ "코드 예시 없이 개념만 설명하지 마시오"
✅ "마케팅 용어 과다 사용 금지"
```

- 틀리면 어떤 종류의 피해가 생기는지 명시
- 금지 조건으로 품질 하한선 보장

### 5. 모델의 행동 지정 (Behavior)

```markdown
역할: AI가 어떤 방식으로 작업을 수행할지 지시

예시:
✅ "단계별로 사고 과정을 보여주며 답변"
✅ "먼저 질문의 핵심을 파악한 후 답변 시작"
✅ "불확실한 부분은 명시적으로 표시"
```

- 사고 방식, 접근 방식 지정
- 검증, 확인 절차 포함

## 5가지 역할 요약표

| 역할 | 영문 | 핵심 질문 | 효과 |
|------|------|-----------|------|
| 범위 지정 | Target Scope | 어디서 찾을까? | 문제 공간 축소 |
| 목적 고정 | Goal | 무엇을 달성할까? | 방향성 확보 |
| 형식 강제 | Format | 어떻게 출력할까? | 구조화된 결과 |
| 오류 금지 | No-Go | 무엇을 피할까? | 품질 하한선 |
| 행동 지정 | Behavior | 어떻게 작업할까? | 과정 품질 |

## 메타프롬프팅의 중요성

### 왜 AI에게 프롬프트를 쓰게 하는가?

> "도메인 키워드는 단순 키워드 기반 생성이 아니다"

AI에게 프롬프트를 작성하게 하면 다음이 자동으로 포함된다:
1. **업계에서 쓰는 개념과 정의**
2. **관행적인 산출물 구조**
3. **좋은 결과를 가르는 평가 기준**

### 단순 키워드 vs 완전한 로직

```markdown
❌ 단순 키워드만 사용:
"SEO 최적화 블로그 글"

✅ 제대로 된 로직 포함:
"SEO 최적화 블로그 글
- 타겟 키워드: [X]
- 검색 의도: 정보 탐색형
- 구조: H1 > H2 > H3 계층
- 메타 설명: 150자 이내
- 내부 링크: 최소 3개"
```

> "어떤 단어를 써야 모델의 문제 공간을 좁힐 수 있는지"를 알아야 한다

## 작업 지시서 vs 질문

### Don't Ask, Specify

| 나쁜 예 (질문) | 좋은 예 (작업 지시서) |
|---------------|---------------------|
| "이거 내 보고서인데 잘 좀 써줘" | "보고서 작성, 상사 보고용, 형식: 요약-본문-결론" |
| "좋은 이메일 알려줘" | "고객 응대 이메일, 클레임 처리, 톤: 공손하되 단호" |
| "코드 작성해줘" | "Python 함수, 입력: JSON, 출력: DataFrame, 예외처리 포함" |

**핵심**: 완벽한 문장보다 **명확한 키워드 10개**가 더 강력하다

## 프롬프트 체크리스트

프롬프트 작성 시 5가지 역할 점검:

```markdown
□ 범위 지정: 탐색 범위가 명확한가?
□ 목적 고정: 성공 조건이 정의되어 있는가?
□ 형식 강제: 출력 구조가 지정되어 있는가?
□ 오류 금지: 하지 말아야 할 것이 명시되어 있는가?
□ 행동 지정: 작업 방식이 지시되어 있는가?
```
