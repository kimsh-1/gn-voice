# 프롬프트 쿠튀르: 전문가의 AI 활용법

> "같은 AI를 쓰는데 왜 전문가가 더 잘 쓸까?"

## 핵심 질문

강의장에서 자주 받는 질문 중 하나:
- "분명 같은 AI를 쓰는데 왜 저 사람은 더 잘 나옵니까?"
- 도메인 지식이라기엔 같은 도구니 비슷하게 나와야 하는데...
- "역시 전문가"라고 표현하기엔 설명이 부족

**답**: 단순히 "프롬프트 잘 쓴다"가 아니라, **단어 선택**이 성능 차이를 만든다.

## 핵심 개념 요약

### 1. 프롬프트 쿠튀르란?

**"당신의 언어는 좌표다"**

- AI에게 전문 용어를 쓰면 → 그 단어가 속한 "똑똑한 데이터 영역"으로 좌표 이동
- 일반인의 "자세히 설명해줘" vs 전문가의 "상세 명세"
- 문제 공간(Problem Space)을 좁히는 것이 핵심

### 2. 왜 전문가가 더 잘 쓰나?

| 일반 사용자 | 전문가 |
|------------|--------|
| "자세히 설명해줘" | 업계 용어로 범위 지정 |
| 넓은 들판에서 평균적 답변 | 좁은 문제 공간에서 정확한 답변 |
| 완연한 문장 + 부사 | 명확한 키워드 조합 |
| 한 번에 완성하려 함 | 생성→검토→수정 루프 |

### 3. 단어가 강력한 이유

> "키워드가 강력한 이유는 모델이 그 단어를 좋아해서가 아니라, 그 단어가 **문제 공간을 좁히는 역할**을 수행하기 때문"

- 전문 용어 = 정보의 좌표
- 쓸데없는 작업이 줄어든다
- 정보를 더 많이 담아서가 아니라 **정보의 좌표를 더 정확히 찍어서**

## 실천 체크리스트

- [ ] 완연한 문장보다 명확한 키워드 사용
- [ ] "알아서", "잘", "적당히" 같은 모호한 부사 제거
- [ ] 목적, 범위, 형식, 금지사항 명시
- [ ] 생성 → 검토 → 수정 루프 적용
- [ ] 결과물을 미리 구상하고 프롬프트 작성
