# 이미지 프롬프트 심화이론

## 개요

나노바나나 Pro 글 중 "JSON 쓰면 잘할 수 있어요, JSON이 정답입니다"라는 말을 많이 보셨을 것입니다.

**핵심 발견**: 일반적으로 많이 쓰이는 프롬프트 중 자연어, 마크다운, JSON, YAML이 **모두 다른 결과**를 낸다는 것

이거 알아두면 특히 **인포그래픽 만드는 데 도움**이 많이 됩니다.

## 프롬프트 형식의 종류

### 5가지 주요 형식

1. **자연어** - 그냥 편히 말로 쓰는 것
2. **마크다운** - 마크업 언어, #으로 구분
3. **JSON**
4. **XML**
5. **YAML**

이 다섯 가지가 현행 가장 많이 쓰이는 양식입니다.
[코드 기반 양식이나 여러 종류가 존재하기는 함]

### 직접 작성할 필요는 없음

**중요**: 이걸 직접 작성하는 게 아니라 **AI한테 작성해 달라고 하면 상당히 잘 작성해줍니다**.

배치는 나중에 생각한다 해도 이러한 특징을 어떤 식으로 활용해야 할까요?

## 각 형식의 특징 (역추론)

### 1. 자연어
- 그냥 입력함
- 가장 직관적

### 2. 마크다운, XML
- **구조화 잘하고 명백하게 계층을 나눔**
- 구분된 섹션 만들기에 유리

### 3. JSON, YAML
- **큰 틀을 묶어주고 특징을 강조**
- 연결된 느낌

**결론**: 구조화 기법에 따른 차이입니다.

## JSON 프롬프트

### 예시 구조

```json
{
  "subject": "premium minimalist coffee machine",
  "materials": ["brushed steel"],
  "features": ["geometric lines", "curved spout", "illuminated touch control panel"],
  "environment": {
    "surface": "white marble countertop"
  },
  "presentation": "floating",
  "technical": {
    "lighting": "studio lighting from top-left",
    "quality": "8K"
  },
  "style": "commercial product photography"
}
```

### JSON의 핵심 포인트

- **키-값 구조**로 파라미터화
- 자동화에 최적
- 스키마 검증으로 **재현성 강화**
- **배치 생성**과 **A/B 파라미터 튜닝**(예: lighting, focus)에 유리
- 조건에서 JSON은 **속성 강제력**이 높아
- 디테일 충실도와 반복 생성 시 **일관성**을 보장하기 좋음

## XML 프롬프트

### 예시 구조

```xml
<image[[prompt]]>
  <subject>premium minimalist coffee machine</subject>
  <materials>brushed steel</materials>
  <features>
    <item>geometric lines</item>
    <item>curved spout</item>
    <item>illuminated touch control panel</item>
  </features>
  <environment>
    <surface>white marble countertop</surface>
  </environment>
  <presentation>floating</presentation>
  <technical>
    <lighting>studio lighting from top-left</lighting>
    <quality>8K</quality>
  </technical>
  <style>commercial product photography</style>
</imagePrompt>
```

### XML의 핵심 포인트

- **태그 기반**으로 의미·영역 구분이 명확
- **레거시/엔터프라이즈 연동**에 유리
- **엄격한 구조** 덕분에 파싱·검증·정책 적용이 용이
- 재현성과 일관성 강화

## YAML 프롬프트

### 예시 구조

```yaml
config:
  scene: "Autumn Harvest Premium"
  subject: "Basket of red and yellow apples"
  style: "Rustic details, cozy"
  imperfections: "natural"
  aspect_ratio: "16:9"
  render: "realistic"
  lighting: "soft"
```

### YAML의 특징

- **가독성**이 좋음
- 설정 파일이나 **시스템 프롬프트 구성**(System Prompt Configuration)에 많이 사용되는 형식

## 실전 비교: 타임라인 인포그래픽

### 자연어 프롬프트
- **연결된 느낌**
- 전체가 이어지는 타임라인

### JSON 프롬프트
- **연결된 느낌** (자연어와 유사)
- 매끄러운 흐름

### Markdown/XML 프롬프트
- **명백한 구분**
- 각 섹션이 확실하게 나뉨
- 구간별로 끊어지는 느낌

### YAML 프롬프트
- JSON과 유사하게 **연결하려는 경향성**

## 실전 활용 가이드

### 구분된 섹션이 필요할 때

> "구분된 섹션을 나눠서 만들 때 저는 **XML이나 마크다운**을 선호합니다."

**예시**: 인포그래픽에서 명확하게 구획을 나누고 싶을 때
- XML 또는 Markdown 사용
- 각 섹션이 확실히 구분됨
- 계층 구조가 명확함

### 연결된 느낌이 필요할 때

**예시**: 매끄러운 타임라인, 연속된 스토리
- JSON 또는 YAML 사용
- 전체가 이어지는 느낌
- 유기적인 연결

### 제품 사진 비교

**JSON 결과**:
- 전체적으로 **둥근 느낌** 강조
- **연결**이 잘 되어 있음

**XML 결과**:
- 확실하게 **각지고**
- **끊어지는** 느낌

## AI 편향성 이해하기

### 편향성 예시: 고양이 사진

고양이 사진만 100장 정도 만들어봤는데:
- **머리의 털 무늬**를 설정 안 해주면
- 다 비슷한 형태로 보이게 됨
- 이것이 AI의 편향성

### 프롬프트 형태에 따른 편향성

**핵심**:
> "프롬프트 형태에 따른 편향성도 존재한다"

**왜 굳이 구분해서 써야 하나?**
> "JSON, XML이 나누려고 하는 경향성을 프롬프팅으로 충분히 이겨낼 수 있다고 하더라도, **그냥 편하게 만들면 나오는 걸 복잡한 길로 갈 필요가 없다**는 것이죠"

### 비유

- 소 잡는 칼로 닭을 잡을 필요가 없고
- 굳이 중식도로 사과를 썰 필요가 없는 것

## 인포그래픽 심화 팁

### 섞어 쓰기 (고급)

"실제 인포그래픽 심화 생성에서는 **섞어서 쓰는 걸 추천**드리지만 그렇게까지는 어려우니"

### 초보자 제언

**계층화하여 명백한 구분의 경우**:
- XML
- Markdown

**아니면 다른 것들**을 써봐라

## 실전 적용 예시

### 명확한 구획이 필요한 인포그래픽

위에 있는 글(작성자가 공개해둔 프롬프트 중 일부):
- **마크다운이나 XML**이 압도적으로 유리
- 자연어-JSON-YAML보다 구분이 명확

### 연표/타임라인

- **자연어-JSON**: 연결된 느낌
- **Markdown-XML**: 명백한 구분

이 정도의 특징만을 이해해도 어렵지 않아요!

## 핵심 정리

### 형식별 특징

| 형식 | 특징 | 적합한 용도 |
|------|------|------------|
| **자연어** | 직관적, 자유로움 | 일반적인 이미지 생성 |
| **Markdown** | 계층 구조, 구분 명확 | 구획이 나뉜 인포그래픽 |
| **JSON** | 파라미터화, 재현성 | 배치 생성, A/B 테스팅 |
| **XML** | 엄격한 구조, 명확한 구분 | 엔터프라이즈, 구획 인포그래픽 |
| **YAML** | 가독성, 설정 | 시스템 프롬프트, 설정 파일 |

### 핵심 인사이트

1. **형식에 따라 결과물이 달라진다**
   - 같은 내용도 형식에 따라 다른 느낌

2. **편향성을 이해하고 활용하라**
   - AI는 각 형식마다 다른 경향성을 보임
   - 이를 이해하고 목적에 맞게 선택

3. **도구를 목적에 맞게 사용하라**
   - 소 잡는 칼로 닭을 잡지 말 것
   - 간단한 방법이 있으면 그걸 쓸 것

4. **AI에게 작성을 맡겨라**
   - 직접 작성할 필요 없음
   - AI가 상당히 잘 작성해줌
