# 나노바나나pro 마스터 하기

## 개요

최근 나노바나나 관련해서 3일내내 연습을했습니다. 영상부터 인포그래픽 포스터까지 다 해봤죠

이제 마지막으로 **뉴스레터용 상세페이지자료**를 만들어 봤습니다.

나노바나나 인포그래픽의 극한활용법이겠네요 **실무에서 쓸수있는 프롬프트**를 배워봅시다.

한번 배워보시죠

## 최종 결과물

### 메인 이미지 1 - 헤더

**주제**: "이제 야근은 AI가 합니다"

**내용**:
1. **OpenAI 오퍼레이터**: 예약부터 결제까지
2. **Google 자비스**: 브라우저 속 비서
3. **Anthropic 컴퓨터 유즈**: 마우스 조작 자동화

**핵심 메시지**: "지시는 사람, 실행은 AI"

### 메인 이미지 2 - 상세 설명

**주제**: "채팅을 넘어 '행동'하는 AI의 시대"

**1. OpenAI 오퍼레이터**
- 복잡한 예약, 사이트 접속부터 결제 직전까지 알아서 척척

**2. Google 자비스**
- 크롬 브라우저 내에서 검색, 쇼핑, 버튼 클릭까지 대신 수행

**3. Anthropic 컴퓨터 유즈**
- 마우스와 키보드를 직접 조작해 지겨운 반복 업무 자동화

**결론**: "당신은 전략을 지시하고, AI는 실행합니다."

## 1. 프롬프트 구조 - 복잡하지만 효과적

### 구조 개요

프롬프트가 좀 길고 복잡합니다.

쉽게말하면:
- **헤더** - 바디 - 세션을 나눠주고
- 맨마지막에 **어떤걸 넣느냐**의 구조인데요

이전글에서 보여드린 **레이아웃 배치를 극한까지 활용**한 녀석이라 프롬프트가 좀 길고 복잡합니다.

### 핵심: 마크다운 활용

**이번녀석은 구조화가 핵심**이기에 **마크다운을 사용**했어

### 프롬프트 예시

```markdown
Create high-quality, vertical layout infographic

**Text Instructions:**
- clean, isometric 3D
- glowing robotic hand taking computer mouse from human

**Header (Top):**
- **Text (Big Bold):** "이제 야근은 AI가 합니다"

**Section (Middle - Divided into distinct sections vertically):**

**Section 1 (Top):**
- **Visual:** airline ticket and calendar being organized
- **Text:** "1. OpenAI 오퍼레이터"
- **Sub-text:** "예약부터 결제까지"

**Section 2 (Middle):**
- **Visual:** browser window (Chrome style) with blue shopping cart
- **Text:** "2. Google 자비스"
- **Sub-text:** "브라우저 속 비서"

**Section 3 (Bottom):**
- **Visual:** spreadsheet and data block being dragged
- **Text:** "3. Anthropic 컴퓨터 유즈"
- **Sub-text:** "마우스 조작 자동화"

**Section 4 (Bottom):**
- simple icon pointing forward and an AI running
- **Text:** "지시는 사람, 실행은 AI"

**Style Parameters:**
- Soft blue, purple, and mint pastel gradients
- Professional, futuristic, clean, readable
- the Korean text spelled correctly as requested
```

## 2. 목적 기반 제시 - Chain 프롬프팅

### 플로우 설계

**Chain 프롬프팅**을 사용한건데:
- **뉴스조사** → **프롬프트작성**의 플로우를 사용할겁니다

### 단계 1: AI 뉴스 조사

맨처음에는 나노바나나로 **카드뉴스 만들껀데 오늘의 AI 뉴스를 조사**시켰어요

**임펙트 위주의 조사**를 진행하였고 결과물은 나쁘지않지만 **아웃풋 포맷 자체를 표로 주려는 문제**가 발생했습니다.

### 조사 결과 데이터

**완료된 Input Data** (복사해서 프롬프트에 이 내용을 프롬프트의 (Input Data) Markdown 섹션에 넣으시면 됩니다)

#### AI 에이전트 시대: 채팅이 아닌 '행동'으로

**1. OpenAI 오퍼레이터 (Operator)**
- 사이트 접속부터 항공권 예약, 결제 직전까지 알아서 처리

**2. 자비스(Jarvis) - 구글의 크롬 브라우저 에이전트**
- 키워드만 입력하면 코딩이나 쇼핑 사이트에 들어가서

**3. Anthropic 컴퓨터 유즈 (Computer Use)**
- 지시가 들어오면 야기가 하나하나 파일을 찾고, 초록색 버튼을 클릭해서

### 단계 2: 프롬프트 구조화

완성된 나노바나나 전문 카드뉴스 제작 전문가 입니다.

본격적인 AI 트렌드 전문 뉴스를 바탕으로, 인스타그램/게시물용 카드뉴스를 친근하게 만들어야 합니다.

**미니멀한 디자인 + 준수하여 Markdown 프로 아이콘 스타일** 주세요.

## 3. 이미지 편향성 문제 해결

### 문제점

**이건 대표적인 이미지 편향성**인데:
- 한국어나 언어표현을 잘못하니 **글을 나중에 추가해야한다**
- 이런정보가 학습되어있어서 **무조건 인포그래픽을 만드려해요**

### 해결 방법

그래서 **오류를 잡아야합니다**:
- 세부적으로 프롬프트로 만들꺼고
- **카드뉴스로 바로 출력되게** 한국어를 쓴다

근데 보면 요약을 많이했죠 이게싫어서 **특정부분을 길게하라 명령**했어

### 수정된 프롬프트 구조

생각하는 단장으로 만들까니까 각 장별로 프롬프트를 가지고왔습니다.

조사한 내용을 바탕으로, 프롬프트를 준비했습니다.

**단독 이미지(Single Sheet)** 형태로:
- 인터페이스, 미니멀한 디자인
- 이미지 생성 툴에 입력하시면 됩니다
- 미드저니(MJ), 달리(DALL-E) 상세하게 복사해서

#### 각 섹션별 이미지 프롬프트

**1. OpenAI 오퍼레이터**
```
Isometric render of sleek, translucent robotic hand hovering over desk setup,
holding mouse and keyboard.
```

**2. Google 자비스 (브라우저 속 쇼핑 비서)**
```
Isometric render focusing on computer screen interface.
Green light (파란, 빨강, 초록)의 Markdown
Isometric of web window interface, resembling Google Chrome,
```

**3. Anthropic 컴퓨터 유즈**
```
Isometric representing workflow with applications.
```

**5장 (Insight): "당신은 전략을, AI는 실행을"**
- 사람은 화면으로 전략적인 그래프를 보고
- 레이아웃 협업의 Markdown

## 6. 실무 적용 팁

### 1. Chain 프롬프팅 활용

**단계별 접근**:
1. **조사**: AI 뉴스나 콘텐츠 조사
2. **구조화**: Markdown으로 계층 구조 설계
3. **생성**: 이미지 생성 툴에 입력

### 2. 마크다운 구조화

**왜 마크다운인가?**
- 계층 구조 설계가 필수
- 자연어보다 명확한 레이아웃 지정
- 일관된 결과물 생성

### 3. 이미지 편향성 대응

**문제**:
- AI가 한국어 텍스트를 잘못 처리
- 기본적으로 인포그래픽 형태로만 생성하려 함

**해결**:
- 세부 프롬프트로 명확하게 지시
- "카드뉴스" 형태로 직접 명시
- 특정 부분을 길게 쓰도록 강제

## 핵심 요약

### 나노바나나 마스터 3단계

**1단계: 목적 설정**
- Chain 프롬프팅으로 조사 → 생성 플로우

**2단계: 구조화**
- Markdown으로 계층 구조 설계
- 헤더, 바디, 세션 명확히 구분

**3단계: 편향성 제어**
- 이미지 생성 툴의 한국어 편향성 인지
- 세부 프롬프트로 명확하게 지시
- 카드뉴스 형태 직접 명시

### 실무 활용 핵심

> **"구조화 안되면 아무것도 못만든다"**
>
> **"시스템화하는게 훨씬 좋다"**

- Structured Output으로 일관성 확보
- 템플릿화로 업무 효율 극대화
- 마크다운으로 계층 구조 명확히
