# UI 잔재 strip 규칙

Threads 상세페이지를 innerText로 스크레이핑한 원본(`-raw-42posts.jsonl`)과
그 원문을 사람이 옮겨적은 카테고리 md(`gongnyangi-threads-db/*.md`)에는 화면 UI 요소가
본문 텍스트에 섞여 들어와 있다. 아래 규칙은 **UI 잔재만** 제거하며, 저자가 실제로 쓴
문장(오타·붙여쓰기·비문 포함)은 한 글자도 고치지 않는다. 각 규칙은 "줄 단위 완전일치"
또는 정규식 매치를 기준으로 하며, 본문 중간에 낀 일반 문장을 실수로 지우지 않도록
개별 라인(스트립된 줄 자체)이 그 조건에 정확히 부합할 때만 제거한다.

## 1. 헤더(글머리) 잔재 — 텍스트 블록 맨 앞에서 제거

작성자 핸들, 배지, 상대시각/절대날짜가 실제 본문 앞에 반복 출력된다.

| 패턴 | 정규식 / 조건 | 예시 |
|---|---|---|
| 작성자 핸들 반복 줄 | 그 블록의 작성자 핸들과 완전히 같은 줄 | `` |
| "AI Threads" 배지 | 완전일치 `AI Threads` | |
| "고정됨" 배지 | 완전일치 `고정됨` | 고정 게시물 표시 |
| 상대 시각 | `^\d+(분\|시간\|일\|주\|개월\|년)( 전)?$` | `39분`, `4일`, `1일` |
| 절대 날짜 | `^\d{4}-\d{2}-\d{2}$` | `2026-06-16` |
| 구분점/작성자 배지 | 완전일치 `·` 또는 `작성자` | 답글 상세에서 "이 글의 작성자" 표시 |
| "방금" | 완전일치 `방금` | |

## 2. 말미(꼬리) 잔재 — 텍스트 블록 끝에서 제거

좋아요/댓글/리포스트/인용 카운터, 정렬·액션 버튼 라벨, 답글창 placeholder.

| 패턴 | 정규식 / 조건 | 예시 |
|---|---|---|
| 숫자만 있는 줄(카운터) | `^\d+(\.\d+)?(천\|만)?$` | `21`, `8`, `1.3만` |
| "N / M" 스레드 위치 표시 | 위 숫자 패턴과 함께 나타나는 `/` 단독 줄 | `1` / `/` / `6` |
| 정렬 라벨 | 완전일치 `인기순` | |
| 활동 로그 라벨 | 완전일치 `활동 보기` | |
| 번역 라벨 | 완전일치 `번역 보기` | |
| 액션 버튼 라벨 | 완전일치 `좋아요` / `답글 달기` / `공유하기` | |
| 무답글 placeholder | 완전일치 `아직 답글이 없습니다` | 답글이 0건인 포스트 말미 |
| 답글 입력창 placeholder | `.+님에게 답글 남기기.*` | `님에게 답글 남기기...` |

## 3. 본문 중간 잔재 — 세그먼트 경계에 낀 것

여러 세그먼트로 이어진 스레드(연속 게시)를 하나의 DOM 캡처가 통째로 물고 올 때,
"N / M" 위치 표시가 문단과 문단 사이에 그대로 낀다. 좌우가 모두 실제 문장인 숫자
단독 줄·`/` 단독 줄은 함께 제거한다(정보성 문장 안의 콤마 숫자, "1.항목" 같이
텍스트가 붙은 번호는 대상이 아님 — 오직 그 줄 전체가 숫자만이거나 `/` 하나뿐일 때).

## 4. 인용 리포스트(타 계정 임베드) 절삭

가 자신의 글 안에 타인의 게시물을 인용(quote-repost)하거나, 자기 자신의
과거 게시물을 인용해 다시 언급하는 경우 원문 캡처에는 그 인용 게시물의 작성자 핸들과
본문이 그대로 이어 붙는다. 이 인용 블록은 특히:
- 타인 작성 리포스트는 `리포스트-제외분.md` 규정상 코퍼스에서 전면 배제 대상이므로,
- 자기 자신의 다른 게시물 인용은 이미 별도 T-문서로 존재하거나 별개 맥락이므로,

**해당 텍스트 블록 안에서 다른 계정 핸들(코퍼스 내 133개 관측 핸들) 또는 자기 핸들
``가 줄 단독으로 다시 나타나는 지점부터 그 블록 끝까지를 절삭한다.**
(적용: T-006, T-019, T-022, T-026, T-029, T-030 — 상세는 각 T-문서 프런트매터 note 참고)

## 5. 공백 정규화

각 줄 끝에 Threads가 삽입하는 개행 없는 공백(U+00A0, non-breaking space)은 일반
스페이스로 치환 후 줄 끝 공백을 제거한다. 문장 내부 글자·오탈자·붙여쓰기는 손대지
않는다.

## 규칙 개수

총 **17개** 개별 패턴(정확히는: 헤더 7 + 말미 8 + 중간 1 + 인용절삭 1) — 위 표의
행 수 기준. 공백 정규화(5번)는 문자 치환이며 "제거 규칙" 카운트에는 포함하지 않는다.
