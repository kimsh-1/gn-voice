# wm-expository 팩 (슬림 v1.0)

## 레지스터
- 화계: 평서 문어체 `~이다 / ~하다 / ~된다`로 통일. 존댓말·구어·감탄 0.
- 자기지칭: 해당 없음(3인칭 객관). 단, 논증 개설의 결론부에서만 겸양 1인칭 복수("정리해 보려고 한다") 허용.
- 호명: 해당 없음. 독자 호명 금지.
- 낙차: 단락 내 `~이다` 반복 → 추정 지점에서 `~로 보인다 / 추정된다 / 할 수 있다`로 갈아 리듬을 준다.
- 태도: 단정(확정 사실)과 유보(기원·계통·연대)의 균형. 이견 대목은 반드시 헤지.

## 시그니처
| 표현 | 기능 | 빈도 |
|---|---|---|
| 명사 종결 정의문(관형절 수식→명사로 끝냄) | 표제어 정의 | 최상·글머리 필수 |
| 'A는 B이다' + 근거 연쇄 | 규정 후 짧은 근거 부착 | 상 |
| 이칭 열거문 "B, C, D 등으로도 불린다" | 명칭 확장 | 상 |
| 전거 인용문 "『문헌』 권N에 ~라고 전한다" | 근거 특정 | 상 |
| 좁은 의미/넓은 의미 구분문 | 외연 이중 규정 | 중 |
| 번호 단계 요약문 "①~ ②~ … N단계로 정리할 수 있다" | 과정 압축 | 중 |
| 명명·개념화문 "이를 ~라고 부를 수 있다" | 현상→개념 승격 | 중 |
| 전자/후자 대조 규정문 | 두 항 대립 명명 | 중 |

## 리듬
- 문장 길이: 관형절 다중 수식의 긴 정의문과 짧은 근거·소결문을 교차한다.
- 종결 변주: `~이다` 연속 뒤 헤지 종결(`~로 보인다/추정된다`)을 끼워 단조로움을 깬다.
- 줄바꿈: 한 단락 = 한 논점(소주제문→예증→소결), 축 전환은 소제목으로 블록 분절.
- 구두점: 개념·인명·서명 첫 등장에 한자/원어 괄호 병기. 서명 『 』·작품 「 」·강조 ' '. 수치는 단위까지(15.3㎝, 600~700℃, 서기전 7500년).

## 구조 관습
- 도입: 표제어를 한 문장, 명사로 끝나는 정의문으로 규정. "~것이 특징이다/~를 의미합니다" 식 서론 금지.
- 전개: 정의 → 개설(유래·이칭·범위·연구사) → 본론(2~5 소제목) → 의의. 본론 축은 시간(편년)·공간(지역)·논리(넓은→좁은 의미) 중 하나로 통일. (유물: 제작→문양→기능→지역→편년)
- 마무리: '의의와 평가'로 사적 위치를 규정("~을 이해하는 데 중요한 지표가 된다"). 요약 재나열·감정적 마무리 금지.

## 대조페어 (금지 → 허용)
- `보여진다 / 모아진다 / 믿겨진다` → `보인다 / 모인다 / 믿어진다`
- `~에 있어서` → `~에서`
- `~로 인하여` → `~때문에`
- `~에 대한 / ~에 대하여` → `~의 / ~을(를)`
- `중요성을 가진다` → `중요하다 / ~을 지닌다`
- `~하고 있다`(상태 서술) → `~한다 / ~이다`
- `~것이다` 종결 연쇄(단락 3회 이상) → `~이다 / ~하다`로 절반 이상 환원
- `옛 문헌에 따르면`(뭉뚱그림) → `『삼국유사』 권5에 …`(서명·권차 특정)
- 무주어 수동으로 행위자 삭제 → 행위자가 있으면 능동
- 해석 없는 병렬 나열 → 열거 직후 그 목록을 해석하는 한 문장 부착

## Do-NOT (이 장르 전용)
- 가치평가를 본문에 흩뿌리지 말 것 — '의의와 평가'에 몰고 본문은 기술(記述)만.
- 확정 사실에 완충어("아마도", "~라고 볼 수 있을 것 같다")를 붙이지 말 것.
- 기원·계통·연대를 단정하지 말 것 — 학계 이견은 반드시 헤지("이설이 분분한", "분명하지 않다").
- 해석 없는 불릿 나열 금지 — 열거 뒤 해석문 필수.
- 독자 호명·"함께 알아봤습니다" 식 마무리 금지.

## 패턴뱅크 (구조 기술 — 문장은 새 소재로 창작)
- **S1 명사 종결 정의문** (C-expository-008): 관형절 수식을 앞에 겹겹이 쌓고 표제어의 범주 명사로 끝나는 무서술어 한 문장 정의. 표제 항목 첫 줄 전용.
- **S2 규정 + 근거 연쇄** (C-expository-018): "A는 B이다" 규정 → 짧은 근거 문장 1~2개 → "그러므로 ~" 소결. 단정의 권위는 근거 부착에서 나온다.
- **S3 번호 단계 요약문** (C-expository-002): "이 과정을 요약하면 ①~⑤ 등 N단계로 정리할 수 있다" 꼴 — 서사적 과정을 원문자 번호로 압축.
- **S4 기술 + 인과 종결** (C-expository-005): 공정 조건(온도·방식·재료)을 수치로 기술한 뒤 "이로 인해 ~을 띤다/된다" 인과로 닫음.
