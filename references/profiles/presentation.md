# profile: presentation — 프레젠테이션 장표 텍스트 (발표덱·리딩덱)

목적: 일반 글·아이디어를 **슬라이드에 들어가는 글**로 집필/변환. 보이스 강도: 0~1 (장표 본문은 0, 스피커 노트에만 1).
근거 규범: Assertion-Evidence(PSU 통제실험 — 어써션 헤드라인이 이해도·회상 우위), Duarte 30/75단어 상한, 민토 피라미드.

## 용도 분기 (미지정 시 발표용 기본, SUMMARY에 명시)
- **발표용(청중 앞)**: 헤드라인 = 완문 어써션 1개(2줄 이내), 본문 = 키워드 3~5개(장표당 30단어 이내), 상세 서술은 전부 스피커 노트로.
- **리딩덱(결재·회람 장표)**: 개조식(명사 종결 "~함/~임" 통일) 허용 — 단 **장표 상단에 거버닝 메시지 한 줄 필수**, 다층 불릿 금지.

## 로직
1. **피라미드 설계 먼저**: 결론(거버닝 메시지) → 근거 3개 이내 → 증거. 도입 1장은 SCQ(상황→문제→질문).
2. **한 장표 한 메시지**: 인사이트가 둘이면 장표도 둘. 75단어 넘으면 그건 문서지 슬라이드가 아니다.
3. **헤드라인은 주장(완문)으로**: "2024 매출 현황"(토픽형) 금지 → "2024 매출은 신제품이 견인해 18% 성장했다"(어써션형). **헤드라인만 위에서 아래로 읽어도 스토리가 흘러야 한다**(타이틀만 읽기 테스트 — 자기검증 필수).
4. **불릿 병렬성**: 한 장표 안 항목은 동일 품사·동일 구조(명사형이면 전부 명사형). 하위 불릿 금지.
5. **대본 분리**: 슬라이드에 말할 문장을 복붙하지 않는다(청중이 읽으면 발표자 말을 못 듣는다). 서사·비유·공냥 터치는 스피커 노트에.
6. **시각 지시**: 헤드라인의 증거가 될 도표·이미지를 장표마다 한 줄로 제안(`[시각: …]`).

## 출력 포맷 (final.md)
```
# [덱 제목] (용도: 발표용|리딩덱, N장)
## S1 — [어써션 헤드라인]
- 키워드 불릿 …
[시각: …]
(노트: 발표자가 말할 내용 — 여기에만 서사·구어 허용)
## S2 — …
```

## 공냥 터치
- 장표 본문에는 0(격식 유지). 스피커 노트에만: 자문자답 전환, 수치 직삽, 화계 낙차 한 방울.
- 오프닝 장표 헤드라인 하나는 통념 뒤집기·수치 훅 허용(카피 P3/명문 훅 유형 준용) — 단 본문이 갚을 수 있는 약속만.

## 금지
- 토픽형 제목 나열, 개조식+어써션 혼용(한 덱 안에서 통일), 다층 불릿, 장표당 두 메시지, 슬라이드=대본 복붙, 검증 불가 수치 날조.
- verify_style.py 미적용(슬라이드 텍스트는 산문 지표 부적합) — 대신 ①타이틀만 읽기 테스트 ②장표당 단어수 ③불릿 병렬성을 자기검증으로 대체.
