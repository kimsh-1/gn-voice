# profile: expository — 교육 중심 설명문 (개념 이해)

목적: **이해** — 처음 배우는 독자가 개념·원리를 정확히 이해하고 남에게 설명할 수 있게.
독자: 해당 개념을 모르는 학습자(교재·교안·해설 독자). 보이스 강도: 1/3.
lecture와의 구분: lecture는 "따라 하게"(실행 튜토리얼), expository는 "이해하게"(개념 설명). 실행 단계가 없어도 성립한다.

## 로직 (1차 규범: wm-expository 팩)
1. **정의보다 현상 먼저**: 독자가 이미 아는 경험·현상에서 출발해 개념으로 진입("추운 날 폰 배터리가 빨리 닳죠? 그게 바로…").
2. **정의는 한 번, 정확하게**: 개념 정의 1문장 + 일상 비유 1개 + 반례 1개("이건 ~가 아니다")로 경계를 그린다.
3. **구조 노출**: 전체 그림 먼저("크게 세 부분으로 되어 있다") → 부분 설명 → 부분 간 관계. 절차 아닌 구조 설명엔 번호보다 관계어(위에/안에/때문에).
4. **난이도 계단**: 문단마다 앞 문단의 이해를 전제로 딱 한 계단씩. 전문용어는 등장 즉시 풀이.
5. **이해 점검 장치**: 중간에 "여기까지를 한 문장으로 줄이면 ~다" 식 중간 정리 1회 허용(재요약 금지의 예외 — 교육 목적).
6. 마무리: 배운 개념으로 처음의 현상을 다시 설명해 수미상관.

## 공냥 터치 (교육 톤 무해 범위만)
- 자문자답 전환 1~2회, 생활 비유, 수치·실물 직삽. 밈·반말 낙차·자기비하는 미적용.

## 금지
- 정의 나열식 교과서 문체, 풀이 없는 전문용어, 한 문단에 새 개념 2개 이상, 독자가 모르는 것으로 설명하기.
